# MovieWhisper -- 可解释电影推荐引擎 设计文档

## 项目背景

本科毕业生求职展示项目，2-3 周内完成。目标是做一个功能完整、有亮点的机器学习项目，覆盖 ML 完整流程，面试时能讲出技术深度。

## 核心创新点

推荐系统 + 可解释性分析。不只推荐电影，还告诉用户"为什么推荐"，体现对推荐系统原理的深入理解。

## 数据

- **来源**：MovieLens 100K 数据集
- **规模**：约 10 万条评分、943 个用户、1682 部电影
- **优势**：推荐系统领域最经典的公开数据集，数据量小，入门级机器可运行

## 架构

```
数据层 -> 算法层 -> 展示层
```

### 数据层

- MovieLens 数据加载与清洗
- 用户画像构建（偏好类型分布、评分历史）

### 算法层

#### 协同过滤（User-Based）

- 余弦相似度计算用户间相似性
- 找到口味相似用户，推荐他们高分但目标用户未看过的电影
- 可解释性输出："与你口味相似的 X 位用户都给这部电影打了高分"

#### 内容推荐（基于电影特征）

- 用电影类型标签（genre）做特征
- TF-IDF 向量化后计算相似度
- 可解释性输出："这部电影属于【科幻/动作】类型，与你喜欢的《XXX》风格相似"

#### 混合策略

- 协同过滤 + 内容推荐加权融合（各 50%）
- 取 Top-10 推荐结果

#### 可解释性面板（核心亮点）

每条推荐附带：
- 推荐来源标签（"相似用户推荐" / "风格匹配推荐" / "两者皆有"）
- 具体理由文字
- 可视化柱状图展示推荐依据强度

### 展示层（Streamlit）

#### 页面 1：评分页面

- 电影列表（可按类型筛选）
- 用户给看过的电影打 1-5 星
- 评分保存到本地

#### 页面 2：推荐页面

- Top-10 推荐列表
- 每条推荐包含：电影名称、类型、预测评分、推荐来源、可展开的详细理由

#### 页面 3：用户画像页面

- 偏好类型分布（饼图/柱状图）
- 最相似的前 5 位用户（相似度分数）
- 评分历史统计

## 技术栈

- Python 3.10+
- pandas / numpy -- 数据处理
- scikit-learn -- 相似度计算、TF-IDF
- Streamlit -- Web 界面
- matplotlib / plotly -- 可视化图表
- surprise（可选）-- 协同过滤快捷实现
